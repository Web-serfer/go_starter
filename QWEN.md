# QWEN.md - Основные директории для Go-проекта

При разработке проекта на Go, я буду использовать следующую структуру директорий:

## Основные директории:

- `cmd/` - точка входа в приложение
  - `cmd/app/main.go` - основной исполняемый файл

- `internal/` - внутренний код приложения (недоступен для импорта извне)
  - `internal/handlers/` - HTTP обработчики
  - `internal/services/` - бизнес-логика
  - `internal/models/` - структуры данных
  - `internal/repositories/` - работа с базой данных
  - `internal/middleware/` - HTTP middleware
  - `internal/utils/` - вспомогательные функции

- `pkg/` - публичные библиотеки (доступны для импорта извне)

- `configs/` - конфигурационные файлы

- `migrations/` - миграции базы данных

- `docs/` - документация

- `tests/` - тесты

- `scripts/` - скрипты для сборки и деплоя

## Файлы:

- `go.mod` и `go.sum` - модульные файлы Go
- `README.md` - описание проекта
- `Dockerfile` - для контейнеризации (опционально)
- `.gitignore` - файлы, игнорируемые Git

## Принципы:

1. Четкое разделение ответственности между слоями
2. Использование `internal/` для кода, специфичного для проекта
3. Поддержка масштабируемости и тестируемости
4. Следование Go-конвенциям и лучшим практикам
